## 170 — Mechanical repeat-N×-under-load gate for timing/poll tests + generous-ceiling lint

The false-FAIL twin of 169's false-PASS guard (FINDING-46: a tight `wait_for`
ceiling false-FAILed ~50% under load, invisible to a 1× run). Two opt-in
mechanical gates:

- **`run-all.sh --repeat-timing[=N]`** (N default 4, clamp 3–5, env
  `WOW_REPEAT_TIMING_N`) re-runs only the timing-flagged subset
  (`grep -lE 'wait_for|sleep|poll'`) N× and FAILS with `FLAKE: <suite> (k/N)`
  when a suite passes some runs and fails/times-out others (`passes>0 && fails>0`).
  Default `run-all.sh` / `--quick` behaviour is unchanged. Catches flakes that
  recur within N consecutive runs — NOT a concurrent-load harness.
- **`lint-timing-ceilings.sh`** floors every `wait_for <file> <pat> <count> <ceiling>`
  ceiling (the LAST positional — quote-safe) at `WOW_MIN_WAIT_CEILING_S`
  (default 30); an inline `# generous` / `# settle` marker opts a call site out.
  Scopes ONLY the `wait_for` ceiling (no bare-`sleep` clause). Exits 0 on the
  real tree (ceilings are 30/40/60s).
