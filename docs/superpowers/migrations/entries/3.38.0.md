## 167 — GitHub bridge adaptive rate-limit backoff

The poll interval is now adaptive. On an HTTP 403/429 rate-limit response the bridge
parses `Retry-After`/`x-ratelimit-reset` (from `gh api -i`) and exponentially backs
off (capped at 15 min); a periodic `rate_limit` probe widens the cadence when primary
`remaining` is low; healthy cycles shrink it back toward the configured floor. Every
cadence change emits an observable `bridge-status {state, interval_sec, reason}`.
Backoff only ever SLOWS polling — strictly better than hammer-and-degrade. Promotes
backlog 136 (the secondary/anti-burst-limit incident).

### Added
- `plugin/tests/github-bridge-ratelimit-backoff.sh` — behavioral cadence test (429+Retry-After → throttle → recovery).

  raises structured on 403/429; `_probe_network` returns `(ok, remaining, reset)`;
  adaptive `interval` + `bridge-status interval_sec` emit.
