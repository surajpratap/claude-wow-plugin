## 175 — Usage-limit statusline: config-dir snapshot install (fixes blank + project-scope bugs)

Reworks the Story-172 opt-in usage auto-pause statusline install. The old
install re-pointed `.statusLine.command` at the wrapper itself and read the
original from `$WOW_STATUSLINE_ORIGINAL_CMD` (an env var nothing set → blank
render), and its install target was an unscoped path so M could install into a
**project** `.claude/settings.json`, shadowing the user's real statusline.

### Modified

- `scripts/wow-process/statusline-usage-persist.sh` — `--install` now resolves
  the **config-dir** `settings.json` (`${CLAUDE_CONFIG_DIR:-$HOME/.claude}`)
  itself and refuses any out-of-config path; it generates a **self-contained
  snapshot** script (`$CONFIG_DIR/wow-usage-statusline.sh`) that inlines the
  persist logic AND delegates to the captured original (stdout + exit passed
  through unchanged). Also repairs a prior buggy install found in a project
  `.claude/settings.json`. `--uninstall` restores the recorded original (or
  removes `.statusLine` when the recorded original was empty — no blank wrapper)
  and deletes the generated script.
- `commands/_manager-startup.md` — the opt-in install invokes `--install` with
  **no** settings-path argument (the script owns config-dir resolution).
- `tests/statusline-usage-persist.sh` — rewritten around the config-dir snapshot
  contract (config-dir resolution, delegate passthrough, empty-original no-blank,
  project repair, uninstall restore, idempotent re-install).

**Consumer action:** if you previously opted into usage auto-pause and saw a
blank or project-scoped statusline, re-run the opt-in (M re-installs on next
startup) — the repair path cleans up the prior install automatically. Otherwise
`/reload-plugins` + restart peers as usual.

**Follow-up:** the interactive "regenerate snapshot on drift" startup offer is
tracked separately (backlog 239).
