# `3.39.1` → `3.39.2`

## 173 — Layer-E rm+glob guard: structure-aware (over-block fix)

The Layer-E guard (`wow-block-rm-glob-non-manager.sh`) matched `\brm\b` plus any
glob char *anywhere* in a command, over-blocking everyday non-destructive
commands for non-M roles (`git add 'apps/*/k8s'`, `grep 'rm.*x'`, `echo "rm *"`).
It now delegates to a quote-aware, structure-aware detector (`_rm-glob-detect.py`,
`shlex` tokenization) that fires ONLY on a genuine `rm`/`rmdir`/`unlink` (or
`xargs rm`) in **command position** carrying a glob. `find -delete` /
`find -exec rm` — the block message's own recommended escape hatch — are allowed;
`find ... | xargs rm` still blocks. Fork-bomb safety is preserved: a tokenizer
parse failure falls back to a command-position-anchored regex biased to block.

The companion M-exemption repair (role resolution via `--git-common-dir` so
`whats-my-role.sh` finds the main-repo markers from a worktree cwd) shipped in
**3.39.1** as the reconciled `${WOW_ROOT:-<git-common-dir>}` form (stories 174 +
173 jointly own line 17); 3.39.2 inherits it.
