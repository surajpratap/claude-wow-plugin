## 159 — Bug tracking as a first-class WOW feature with a standard schema

`implementations/bugs/*.md` becomes a first-class WOW artifact with a standard HTML-comment-marker schema, a status state machine, and helper scripts that are the only authorized writers of the markers. A validator enforces the schema in the test suite; T/PP/SD/M doctrine flows are routed through helpers instead of hand-edits.

### Added

- `plugin/scripts/bug-shape-check.sh` — validator. Walks every `implementations/bugs/*.md`; asserts required markers per current status, enum values, id-vs-filename match, ISO-8601 timestamps, `duplicate-of` references. Exit 0 on all-pass; non-zero with per-file diagnostics. Wired into `plugin/tests/run-all.sh` (plugin-only — root `tests/run-all.sh` stays scoped to source-repo artifacts).
- `plugin/scripts/bug-emit.sh` — file generator. Takes `--reporter --severity --priority --affected-story --affected-version --title` flags. Picks next 4-digit ID atomically via `fcntl.flock` on `implementations/bugs/.id-allocation.lock` (concurrent T sessions can't collide). Writes the file with all `filed`-required markers + empty body section headers; prints path on stdout.
- `plugin/scripts/bug-state-transition.sh` — atomic state changer. `<bug-id> <new-status> --agent-id <id> [--pr-url <url>] [--fixed-in <v>] [--duplicate-of <id>] [--reason <text>]`. Refuses illegal transitions with stderr diagnostic. On success updates required markers + appends a `## State log` line; on `fixing`/`fixed`/`closed` transitions auto-emits the corresponding bus message via the MCP CLI.
- 10 new bash tests at `plugin/tests/`: `bug-shape-check-{happy,missing-required,bad-enum,duplicate-ref,id-mismatch}.sh`, `bug-emit-happy.sh`, `bug-state-transition-{happy,illegal,missing-required}.sh`, `migration-existing-bugs-conform.sh`.

Suite count: +10 (resolved at merge time relative to whatever the post-merge baseline is).

### Modified

- `plugin/commands/_agent-protocol.md` — new `## Bug schema` section before `## Bug lifecycle`. Defines required markers per status, enums, state machine, default severity/priority decision tree, helper-only authorship rule, state-log convention. Existing `## Bug lifecycle` rewritten to invoke helpers (no more hand-edit prose).
- `plugin/commands/tester.md` — `bug-found` flow invokes `bug-emit.sh`; `bug-fixed` re-verify uses `bug-state-transition.sh <id> verified`.
- `plugin/commands/pair-programmer.md` — `bug-verified` handler invokes `bug-state-transition.sh <id> triaged`.
- `plugin/commands/senior-developer.md` — `bug-triaged` handler invokes `bug-state-transition.sh <id> fixing` then `<id> fixed --pr-url --fixed-in`.
- `plugin/commands/manager.md` — `bug-found` no longer flips status (PP does that via the helper). M-driven `wont-fix` + `closed` use the helper.
- `plugin/tests/run-all.sh` — adds `bug-shape-check.sh` invocation (fail-fast on malformed bug, same pattern as `plan-shape-gate.sh`).

### Consumer action

`/reload-plugins`. New behavior takes effect on next role startup.

**If a consumer project has bug files using non-conformant status enum values** (e.g., legacy `reported`, `open`, `wip`, `in-progress`), running `bash "$(wow-locate scripts/bug-shape-check.sh)"` post-upgrade will fail with per-file diagnostics. Remediate with a one-shot manual edit: map each non-conformant status to the closest canonical enum value:

| Legacy   | Canonical  |
|----------|-----------|
| `reported` | `filed`   |
| `open`     | `triaged` |
| `wip`, `in-progress` | `fixing` |
| `resolved`, `done` | `fixed` or `closed` (per workflow state) |

Then add any newly-required markers per the schema (`id`, `reporter`, `reported-at`, `severity`, `priority`, `affected-story`, `affected-version` at minimum). Future bug emit/transition uses the canonical enum mechanically.

The `<!-- status: ... -->` HTML comment marker is the canonical state source-of-truth. Hand-editing markers is forbidden going forward — `bug-emit.sh` and `bug-state-transition.sh` are the only authorized writers.
