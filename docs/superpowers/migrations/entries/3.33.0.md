## 158 ‚Äî Memory ‚Üî learnings consolidation

CC has its own auto-memory system that agents sometimes write to instead of `implementations/learnings/<role>.md`. This release adds a mechanical consolidation step that periodically migrates relevant memory entries into the role's durable learnings file with provenance, dedup, and role attribution.

### Added

- `plugin/scripts/consolidate-memory.sh` ‚Äî POSIX bash script. Walks `${CLAUDE_CONFIG_DIR:-$HOME/.claude}/projects/<encoded-project>/memory/` (encoded = `$(git rev-parse --show-toplevel) | sed 's|/|-|g'`). 4-path attribution heuristic (frontmatter `metadata.role`, body `[role: X]` marker, exactly-one-role mention, filename prefix). Atomic-write on learnings; appends ambiguous entries to a triage file. Stdout JSON summary `{role, path, entries_added, entries_skipped, triage_count}`. `WOW_DROP_CONSOLIDATED_MEMORY=1` env deletes memory file after consolidation (default keeps with `consolidated-into:` frontmatter marker).
- `plugin/scripts/startup/phase_memory_consolidation.sh` ‚Äî startup-phase library invoked **immediately BEFORE `bootstrap`** by `startup.sh` for every role. mtime-check the role's memory dir vs learnings file; on hit, invokes the consolidation script + emits `info` action lines. Never fails startup ‚Äî a hiccup degrades to a logged skip.
- `plugin/commands/_meta/consolidate-memory.md` ‚Äî slash command `/consolidate-memory` invokes the script for the active role.
- `plugin/tests/consolidate-memory-{happy,idempotent,attribution-heuristic,ambiguous-triage,dedup,drop-flag,startup-trigger,startup-no-trigger,bus-emit}.sh` ‚Äî 9 new bash tests covering the script + startup phase + doctrine grep.
- `plugin/tests/mcp-server-allowed-types-includes-learnings-consolidated.sh` ‚Äî assertion that the new MCP type is registered (Story 087 test pattern).
- New MCP type **`learnings-consolidated`** in `mcp/claude-wow-server/server.py` `ALLOWED_TYPES`. Payload: `{role, path, entries_added, entries_skipped, triage_count}`. Distinct from existing `learnings-updated` (different payload shape, different trigger surface). Documented in `_agent-protocol.md`.

Suite count: 169 ‚Üí 179.

### Modified

- `plugin/scripts/startup.sh` ‚Äî `phases_for_role()` slots `memory_consolidation` immediately BEFORE `bootstrap` for every role (manager `env layout version sweep coherence peer memory_consolidation bootstrap`; the others `env memory_consolidation bootstrap`). Required because `bootstrap` emits `complete` ‚Äî consolidation must precede so `info` lines land before the agent is told startup is done.
- `plugin/commands/_retro-doctrine.md` ‚Äî Learnings-refresh window step 2 adds: M emits a `nudge` (to `*`) with `payload: {repair: "consolidate-memory"}` at the start of the window. Each peer's existing `nudge` handler routes the `repair` payload to invoke the script + emit `learnings-consolidated`.
- `plugin/commands/manager.md`, `senior-developer.md`, `pair-programmer.md`, `tester.md`, `slacker.md` ‚Äî `nudge` handler tables gain a one-line `payload.repair == "consolidate-memory"` case. Now accept `to: *` for cross-role broadcast nudges (was previously role-glob or own ID only).
- `plugin/commands/_agent-protocol.md` ‚Äî adds `learnings-consolidated` row to the Bus message types table.
- `.gitignore` ‚Äî adds `implementations/learnings/.consolidate-needs-triage.md` (runtime-only triage file).

### Consumer action

`/reload-plugins`. New behavior takes effect on next role startup or sprint-end retro window. No peer restart required ‚Äî the mechanical changes are picked up by `startup.sh` on next invocation; existing sessions continue working with the old startup-phase set until restart.

The `WOW_DROP_CONSOLIDATED_MEMORY=1` env var is opt-in; default OFF keeps a frontmatter marker on consolidated memory files for audit.
