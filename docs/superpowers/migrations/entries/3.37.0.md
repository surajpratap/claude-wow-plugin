## 165 — GitHub bridge conditional requests (ETag / 304)

`_gh_api` now sends `If-None-Match` for the list + 4 per-PR sub-calls, caching the
ETag + last body per `(repo, endpoint)` at `cursor_root/<slug>/etags.json`. A
`304 Not Modified` returns the cached JSON at zero quota cost; the `gh api -i`
status is parsed from stdout (gh exits nonzero on 304), and on any header-parse
ambiguity the bridge falls back to a full GET (correctness over savings).
~90–99% cut on quota-counted steady-state sub-calls. Also hardens the shared
gh-shim status-counter (FINDING-45) so 200→304 conditional tests can't be hollow.

### Added
- `plugin/tests/github-bridge-etag-conditional.sh` — behavioral 200→304 round-trip,
  with a `bridge-status:degraded` distinguisher (a raise-on-304 bug can't pass) and
  a malformed-header → bare-GET fallback case.

### Modified
- `plugin/bridge/github/run.py` — conditional `_gh_api(etag_dir=...)` + ETag cache
  (`_parse_gh_include` / `_read_etag_cache` / `_write_etag_cache`), threaded via a
  `cursor_root` param onto the 3 emit fns + the list call.
- `plugin/tests/fixtures/gh-shim.sh` — FINDING-45 status-counter (per-endpoint dir
  or fail-closed), an `IF-NONE-MATCH` call-log marker, a `WOW_GH_MALFORMED_HEADERS`
  knob, and a `>299 → nonzero exit` semantic (models real `gh`).

Suite count: +1 plugin suite.
