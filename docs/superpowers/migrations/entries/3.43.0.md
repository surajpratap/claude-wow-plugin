## 172 ‚Äî Opt-in mechanical 5h/7d usage auto-pause (statusline persist ‚Üí idle-limit-monitor poll ‚Üí bus directive)

Removes the last routine human-intervention stall ‚Äî Claude Code 5h/7d
usage-window exhaustion ‚Äî with an **OPT-IN, mechanism-first** feature. The
consumer's statusline (the only place a process can observe `rate_limits`)
persists the usage data to a state file; the idle-limit-monitor's 60s loop polls
it and emits a voluntary pause/resume **bus** directive at threshold, with a
time-based reset for the 5h window and a human-escalation for the 7d window.

Nothing fires unless the human opts in: M asks once via `AskUserQuestion` when
the tracker key `usage_autopause` is unset, persists the answer, and on opt-in
installs the statusline persist wrapper. Default (unset) = OFF.

### How it works

- **Delegating statusline wrapper** (`scripts/wow-process/statusline-usage-persist.sh`,
  M-installed on opt-in): reads stdin once, atomically writes the
  `rate_limits` (5h + 7d `used_percentage`/`resets_at`) to a state file, then
  delegates stdin through to the consumer's recorded original statusline command
  unchanged (stdout + exit preserved). Absent `rate_limits` ‚Üí writes nothing.
  Install re-points `statusLine.command` and records the original;
  `--uninstall` restores it (opt-out).
- **idle-limit-monitor limit codepath** (additive to `idle-monitor.py`, runs
  every tick outside the `.nothing_to_do` guard; the idle predicate/wake logic
  is unchanged): polls the state file and on the **5h window `>=98`** bus-emits
  ONE `usage-limit-pause` directive (`directive:"pause"`), writing a pause-marker
  for idempotency. The **daemon itself** (timer-driven, not M's LLM) bus-emits
  ONE `usage-limit-reset` (`directive:"resume"`) once `now >= resets_at` ‚Äî the
  5h pause/resume auto-cycle is fully daemon-driven and never %-gated on resume.
  The **7d window `>=99`** bus-emits ONE `usage-limit-7d-escalate` directive
  (`directive:"escalate"`) addressed **`to: manager-*`** (M-private ‚Äî only M
  acts, peers ignore `escalate`; FINDING-47): M escalates to the human, the team
  is NOT auto-halted (the 7d reset is days away), and any pause is the human's
  post-escalation choice. All three directives are emitted via the sanctioned
  `bus_emit` CLI path (not the M-private `print()`) so the consumer sees them on
  the bus, using a DISTINCT `limit-monitor-<ts>-<6hex lowercase>` producer ID
  (NOT `idle-monitor-`, which M routes to its private idle-event handler before
  type-handling ‚Äî that would misclassify the directive).
- **Bounded, role-asymmetric directive-obey rule** (`commands/_agent-protocol.md`):
  a bus message whose `payload.directive` is in the CLOSED set
  `{pause, resume, escalate}` (exact string equality) is a standing order,
  applied role-asymmetrically ‚Äî the four PEER roles (senior-developer /
  pair-programmer / tester / slacker) halt on `pause`, continue on `resume`, and
  IGNORE `escalate` (M-only); the MANAGER is EXEMPT from `pause`/`resume` (the
  driver/channel/resume-producer ‚Äî M's LLM never halts) but ACTS on `escalate`
  (escalates to the human). Any other directive value is ignored (not arbitrary
  instruction execution). Checked BEFORE the absorb-unknown rule.

### Consumer action

`/reload-plugins` + restart peers. No data migration. The feature is OPT-IN:
nothing changes until the human enables `usage_autopause` (M asks once on
startup) and M installs the statusline persist wrapper. Opt-out via
`statusline-usage-persist.sh --uninstall <settings.json>` restores the original
statusline command.
