## 152 ‚Äî Script the mechanical agent startup

### Added

- **`plugin/scripts/startup.sh`** ‚Äî single entry point for every role's mechanical startup. POSIX bash. `--role <one of: manager | senior-developer | pair-programmer | tester | slacker>` runs the per-role phase plan and emits one JSONL action line per scriptable step on stdout. `--verify` checks every tracker-recorded Monitor's PID is alive (kills -0). `--resume --answer <key>=<value>` re-enters from the per-agent checkpoint file at `.agents/<agent-id>.startup-state.json`.
- **`plugin/scripts/startup/` directory** ‚Äî phase libraries (`phase_env.sh`, `phase_layout.sh`, `phase_version.sh`, `phase_sweep.sh`, `phase_coherence.sh`, `phase_peer.sh`, `phase_bootstrap.sh`) + emit helpers (`lib_emit.sh`) + checkpoint primitives (`lib_checkpoint.sh`). Each phase function is independently testable.
- **`plugin/scripts/migrations/` directory** ‚Äî empty in this story; reserved for future per-version `<from>-<target>.sh` migration scripts that `phase_version` discovers + dispatches.
- **Closed action enum at the function boundary.** `lib_emit.sh` exposes 5 emit functions: `emit_info`, `emit_arm_monitor`, `emit_ask_human`, `emit_complete`, `emit_abort`. There is no `emit_schedule_wakeup` or `emit_start_loop` ‚Äî backlog 190's symptom (SD reaching for a scheduler for bus-tail) is closed by construction.
- **`build_arm_monitor_command` helper** ‚Äî every `arm-monitor` instruction's `spec.command` includes the story-154 `monitor-pipe.sh` downstream pipe + the empty-PIPE fallback. Without monitor-pipe.sh resolvable, falls back to `exec bash <wrap-script>` (unwrapped; CC sees truncated events but the protocol still works).
- **M's preflight transient tracker.** `phase_peer` writes `.agents/manager-preflight-<id>.json` BEFORE emitting pings, removes it via `trap EXIT` after pong-collection. Story 149's dead-agent-ID guard checks for the tracker file's existence ‚Äî it's present during the ping window, so peers can `pong` by exact ID without working around the guard. Closes the 2026-05-27 retro convergence where PP/SD/T all independently fell back to the `manager-*` role-glob.
- **5 new `_<role>-startup.md` files** (~19 lines each) ‚Äî invocation recipe + handoff-protocol pointer. Old content frozen as `_<role>-startup-legacy.md` for one release; deleted in next minor.

### Why

Each role's startup procedure was ~400 lines of prose interpreted by every CC session from scratch. Two concrete drift symptoms drove this: (1) SD reaching for `ScheduleWakeup` / `/loop` for bus-tail instead of reactive `Monitor` (backlog 190); (2) PP/SD/T all independently working around the dead-agent-ID guard (story 149) on M's preflight ping ‚Äî same workaround, 3 separate session re-interpretations.

Scripting the mechanical steps + a closed action enum closes both classes by construction:
- The script literally cannot tell CC to use a scheduler for bus-tail (no such verb in the contract).
- The transient preflight tracker satisfies the 149 guard mechanically; peers respond to the exact preflight ID without fallback.

### Consumer action

`/reload-plugins` + restart all peers per standard upgrade hygiene. Each role's next session runs `bash startup.sh --role <role>` instead of interpreting `_<role>-startup.md` prose. Old `_<role>-startup.md` files are preserved as `_<role>-startup-legacy.md` for one release ‚Äî the next minor will remove them.
