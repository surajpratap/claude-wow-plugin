## 182 — tool-use idle reminder (PostToolUse hook)

A PostToolUse hook on Bash/Read/Write/Edit surfaces a `resume_work` reminder (via `hookSpecificOutput.additionalContext`, delivered as a system-reminder beside the tool result) when the `.nothing_to_do` idle marker is set — catching non-prompt work-resumption (Slack-driven, autonomous) that 179's UserPromptSubmit marker-clear doesn't cover. Reminder-only (never clears the marker); global (all roles); throttled once per marker-episode per session.

### Added

- `plugin/scripts/hooks/wow-tool-use-idle-reminder.sh` — the PostToolUse hook. Silent when the marker is absent or the tool is out of set; throttles via `implementations/.tool-use-reminder/<session>` keyed on the marker's `ts` (a clear+re-set is a new episode → re-reminds).
- `plugin/tests/tool-use-idle-reminder.sh` — 6-case red-green (emit / silent-absent / silent-out-of-set / reminder-only / throttle / episode-reset) + `.red-without/182-tool-use-reminder-throttle.patch`.

### Modified

- `plugin/hooks/hooks.json` — adds the first `PostToolUse` entry (matcher `Bash|Read|Write|Edit`).

**Consumer action:** `/reload-plugins` to load the new PostToolUse hook (running sessions pick it up after reload). Suite count +1 (`tool-use-idle-reminder.sh`). No role-doctrine change.
