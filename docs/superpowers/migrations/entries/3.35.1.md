## 163 тАФ Inert-wiring hotfix batch (bugs 0006/0007/0008/0009)

Post-sprint hotfix for the EMITTED-BUT-INERT bug class M's adversarial completeness audit flagged across 9/11 stories in sprint 2026-05-28-dx-batch. All four fixes are paired with a BEHAVIORAL test (creates state, runs the real script/handler, asserts the observable effect) тАФ NOT a source-grep. Shape-only tests are exactly why every gap shipped; this release inverts the discipline.

### Bug 0006 (P0) тАФ `--exec bus-emit` was the wrong CLI form

`server.py` `__main__` only dispatched the CLI emit path on `sys.argv[1] == "bus_emit"` (positional). Three call sites used `python3 server.py --exec bus-emit "$JSON"` instead тАФ `argv[1] == "--exec"` fell through to the stdio JSON-RPC server, which read stdin to EOF and **emitted nothing**. The 159 bug-lifecycle bus emits + 152 startup hello/ping were silently inert.

Fix: rewrote all three call sites to the positional `bus_emit --from --to --type --payload-json` form (Option A тАФ single CLI source of truth, no `--exec` alias). Removed the `2>/dev/null` swallow that hid the failure mode; emit failures now print a diagnostic on stderr.

### Bug 0007 (HIGH) тАФ monitor-spec.sh bypassed monitor-pipe

`monitor-spec.sh` set `CMD="bash $WRAP_SCRIPT [args]"` per purpose, with **no** `| bash monitor-pipe.sh --purpose <p>` pipe. Story 154's central promise тАФ every Monitor source pipes through `monitor-pipe.sh` for the pointer + read-tool pattern тАФ was violated for the entire post-compact re-arm path; re-armed Monitors emitted truncated raw event text instead of pointers.

Fix: `monitor-spec.sh` now sources `lib_emit.sh` and routes through `build_arm_monitor_command(purpose, extra_args)`. Single source of truth for the pipe-wrap shape тАФ startup-time arming and post-compact re-arm now share one implementation.

### Bug 0008 (HIGH) тАФ Slacker reaction states not wired + lazyReconcile null

Three of five reaction states (`received`, `refusing`, `escalated`) had table entries in `slacker.md` but NO imperative steps in the decision tree тАФ dead doctrine, the human-facing sub-second receipt signal never fired. `ReactionManager.lazyReconcile()` unconditionally returned null; after bridge restart, `setState` saw no previous reaction and stacked instead of replacing.

Fix:
- `slacker.md` decision tree gains explicit `/set-reaction state:<received|refusing|escalated>` imperative steps at the inbound, decline, and human-escalation transitions.
- `reactions.ts` `lazyReconcile()` now lazy-fetches the bot's user id from `auth.test` (cached), queries Slack via `reactions.get`, and returns the emoji whose `users` array includes the bot.

### Bug 0009 (MED) тАФ Slack attachments stale dist + e2e

The `_slacker-startup` build gate rebuilt only on `[ ! -d dist ] || [ LOCK_SHA != SAVED_SHA ]`. Story 157 shipped with the SHAs matching but `src/` edited; the gate skipped, a stale `dist/index.js` ran without the Attachments wiring. The 10 `slack-attachment-*` tests instantiated `Attachments` in isolation; none drove `registerHandlers` end-to-end.

Fix:
- Build gate adds a third clause: `find src -newer dist -print -quit` тАФ any source newer than dist triggers a rebuild regardless of SHA delta.
- Local dist rebuilt one-shot (not committed; `dist/` is gitignored).
- New `slack-attachments-handler-e2e.sh` test drives `registerHandlers` with a real `Attachments` instance, fires a synthetic message event, and asserts the feed `append` call received `attachments:[{path}]`.
