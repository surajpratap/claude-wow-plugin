## 164 ‚Äî GitHub bridge polls `state=open` (rate-limit sprint stack base)

The bridge now lists `pulls?state=open` instead of `state=all`, and never
re-runs the per-PR sub-call fan-out (reviews/comments/check-suites) on
closed/merged PRs. The open‚Üíterminal (closed/merged) `pr-state` transition is
still emitted exactly once ‚Äî detected by the PR's disappearance from the open
list and confirmed by one targeted `pulls/<num>` fetch (which also distinguishes
merged vs closed and avoids a `per_page` page-overflow false-positive) ‚Äî then the
PR's cursor is stamped `finalized` and skipped. A reopened PR is un-suppressed and
resumes fan-out. ~90% steady-state `gh api` reduction on long-lived repos.

By design, late events on already-terminal PRs are no longer polled.

### Added

- `plugin/tests/github-bridge-poll-state-open.sh` ‚Äî behavioral open‚Üímerged‚Üíreopened
  round-trip + per_page-guard case + a harness self-check, driving the real
  subprocess via the gh-shim call-log (no monkeypatch).
- `gh-shim` shared harness (consumed by sprint stories 165/166/167): skips leading
  `-i` **and** repeated `-H <value>` so `gh api -i -H 'If-None-Match: <etag>' <path>`
  dispatches by the real path; `-i` response headers driven by env, including a
  **sequenced** status (`WOW_GH_STATUS_LIST` + `WOW_GH_STATUS_COUNTER` for a

  plus `WOW_GH_ETAG` / `WOW_GH_RETRY_AFTER` / `WOW_GH_RATELIMIT_REMAINING`; a
  per-invocation call-log (`WOW_GH_CALL_LOG`); and a single-PR `*/pulls/<num>`
  detail case (`WOW_GH_PR_DETAIL_FILE`/`_LIST`/`_COUNTER`).
