# `3.45.0` → `3.46.0`

## 177 — Unified release script: deliverable-only artifacts to both targets

Source-repo release tooling change. Consumers see a smaller `dist` tree.

### Added

- `scripts/release.sh` (source-repo root, NOT bundled) — one command that strips
  source-only files (`tests/`, `CLAUDE.md`) on a scratch worktree, then publishes
  the deliverable-only tree to BOTH the nedati `dist` branch (+ tag + GitHub
  release) and the surajpratap fork `main`. `--dry-run` previews; `--stage-only`
  builds the stripped staging ref without pushing. Fork target configurable via
  `WOW_FORK_REMOTE` (default `personal`) / `WOW_FORK_BRANCH` (default `main`).

### Removed

- `scripts/release-dist.sh` — superseded by `release.sh` (the one release entry
  point). References in `manager.md` Phase-7, root `README.md`, and root
  `AGENTS.md` repointed.

### Modified

- `tests/dist-tree-shape.sh` — asserts the STRIPPED release shape (`tests/` +
  `CLAUDE.md` absent, required dirs present) by driving the real producer
  `release.sh --stage-only`, with a self-contained red-green proving the strip is
  load-bearing.

**Consumer action:** none for behavior. The published `dist` tree no longer ships
`tests/` or `CLAUDE.md` — a smaller install, no API change. `/reload-plugins` +
restart peers as usual after updating.
