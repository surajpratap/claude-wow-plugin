## 184 ‚Äî worktree-invariant bus ROOT resolution + runtime guard

`startup.sh` + `bus-tail.sh` resolved ROOT via `git rev-parse --show-toplevel` ‚Äî from a worktree cwd that's the WORKTREE root, so the bus-tail Monitor tailed the worktree-local `.message-bus.jsonl` while `bus_emit` wrote the main-repo bus (the agent emits but doesn't listen ‚Üí M reads it as dead). Now both resolve via the worktree-invariant `--git-common-dir`‚Üíparent idiom (matching `whats-my-role.sh`), plus a runtime inode guard.

### Added

- `scripts/wow-process/bus-tail.sh` `assert_main_bus` guard ‚Äî before the tail loop, refuses (`EXIT_BUS_NOT_MAIN`, non-zero exit) when the handed bus isn't the main-repo bus (a `-ef` device+inode test against the `--git-common-dir`-resolved main bus). Plus a `--check-bus-root <path>` selftest mode (resolve + guard + exit, no tail loop).
- `plugin/tests/bus-root-worktree-invariant.sh` ‚Äî 3-case red-green (resolution-from-worktree, guard-fires, `WOW_ROOT`-override) on a real `git worktree` fixture, + `.red-without/184-resolution-show-toplevel.patch` and `.red-without/184-guard-removed.patch`.

### Modified

- `scripts/startup.sh` + `scripts/wow-process/bus-tail.sh` ‚Äî ROOT resolution uses `$WOW_ROOT` ‚Üí `--git-common-dir`‚Üíparent ‚Üí `pwd` fallback; never `--show-toplevel` for the bus path. bus-tail's ROOT is now computed independently of the handed `$BUS` (so the guard is meaningful, not self-consistent).

**Consumer action:** `/reload-plugins` + restart peers to pick up the worktree-invariant resolution (a running bus-tail re-arms on the correct main bus after restart). Suite count +1 (`bus-root-worktree-invariant.sh`); no role-doctrine change.
