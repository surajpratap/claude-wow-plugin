## 161 ‚Äî Hot-fix bug 0003 (startup.sh BLOCKER + 3 MAJORs in 3.29.0)

P0 hotfix bundling 4 startup.sh bugs PP found post-merge on 3.29.0. Target patch release.

### Fixed

- **FINDING-40 (BLOCKER)** ‚Äî `startup.sh:run_phases` no longer silently succeeds when a phase fails. Logic inversion (`if ! phase_X; then return 0;`) replaced with an explicit case statement: rc=0 continues, rc=10 is a clean ask-human handoff, anything else triggers `emit_abort` + non-zero exit. v3.29.0 agents could complete startup with broken phases ‚Äî they cannot anymore.
- **FINDING-41 (MAJOR)** ‚Äî `phase_peer` no longer installs `trap EXIT INT TERM` to remove the preflight tracker. The trap fired at `startup.sh` exit, BEFORE M's operating doctrine collected pongs, defeating the dead-agent-ID guard fix story 152 was meant to deliver. The tracker now persists until M's operating doctrine cleans it up ‚Äî new "Post-startup preflight cleanup" section in `commands/manager.md` covers the lifecycle handoff.
- **FINDING-42 (MAJOR)** ‚Äî `phase_bootstrap.sh` arm-helper callers now check `$?` (`_emit_bus_tail_arm "$role" "$agent_id" || return 1`), and helpers themselves abort + return 1 on either `build_arm_monitor_command` or `emit_arm_monitor` failure. `emit_complete` is wrapped too ‚Äî jq error or closed stdout now abort cleanly instead of cascading to a misleading `complete`.
- **FINDING-43 (MAJOR)** ‚Äî `startup.sh --verify` now reads `role-process-map.json` for the active role, filters out `?`-suffix optional purposes and the github-bridge when `.github/config.json` is absent. If any required purpose is expected AND the tracker has zero `*_task_id` entries, verify emits `EXIT_NO_MONITORS_ARMED <role> expected: [...]` on stderr and exits non-zero. v3.29.0 agents could `--verify` clean with zero Monitors armed ‚Äî they cannot anymore.
