# `3.46.1` → `3.47.0`

## 179 — Decouple idle from AFK: hook clears the idle marker; AFK stays M-judged

A UserPromptSubmit hook now mechanically clears the `.nothing_to_do` idle marker on every prompt (the mechanical equivalent of `resume_work`), and M no longer treats a prompt as an automatic `/back`/AFK-exit — AFK exit is M's judgment of the message. Idle (team-has-no-work) and AFK (human-away) are now orthogonal concepts.

### Added

- `plugin/scripts/hooks/wow-clear-idle-marker.sh` — UserPromptSubmit hook; idempotent, role-agnostic `.nothing_to_do` clear; never touches AFK state.

### Removed

- `plugin/scripts/hooks/wow-remind-resume-work.sh` — the M-only "remind M to call `resume_work`" hook, obsoleted by the mechanical clear (renamed → `wow-clear-idle-marker.sh`).

### Modified

- `plugin/hooks/hooks.json` — the second `UserPromptSubmit` entry now wires `wow-clear-idle-marker.sh`.
- `plugin/commands/manager.md` — AFK-handling intro + Section G rewritten so a prompt no longer auto-exits AFK (M-judged); idle and AFK documented as orthogonal; autonomous-pickup condition-1 language reconciled.
- `plugin/commands/_agent-protocol.md` — `.nothing_to_do` marker contract now notes the role-agnostic mechanical hook-clear (the `declare_idle`/`resume_work` MCP tools stay M-only); the hook never touches AFK.

The `<user-prompt-submit-hook>` handlers (`last_user_prompt_ts`, disapproval-brake, github-bridge re-arm) are **unaffected** — they key on the user prompt turn, not the removed `additionalContext` emission.

**Consumer action:** `/reload-plugins`, then restart peers via `/<role>` — a running M retains the old prompt-as-implicit-`/back` doctrine and running sessions retain the old hook until restart. Suite count unchanged (the dedicated test is renamed, not added).
