## 162 — Hot-fix bug 0004 (Slack attachments: SSRF + path-traversal in 3.32.0)

P1 security hotfix bundling both fixes from background security-review on PR #168.

### Fixed

- **FINDING-44 (HIGH SSRF / credential-exfil)** — `attachments.ts:download()` now parses `url_private_download` via `new URL(url)`, requires `https:` scheme, and verifies the host is in the hardcoded `['files.slack.com', '.slack.com']` allowlist (exact match for the FQDN, `endsWith` for the workspace-subdomain suffix). On rejection, throws BEFORE sending the Authorization header — `downloadForMessage`'s existing per-file catch produces a `{skipped: true, skip_reason: "non-Slack host: <host>"}` entry. Plus `fetch(..., { redirect: 'error' })` so a Slack-allowlisted URL redirecting to a non-Slack host can't smuggle attacker-controlled bytes into the file write either.
- **FINDING-45 (MEDIUM path-traversal)** — `attachments.ts:downloadForMessage()` validates `messageTs` against `/^\d+\.\d+$/` at function entry; on rejection, logs + returns `[]` without creating any directory. Belt-and-suspenders: per-file `resolve(destPath)` + `startsWith(resolve(this.baseDir) + sep)` check inside `download()` before write — defeats any future regression in either guard.

### Added

- `plugin/tests/slack-attachment-ssrf-guard.sh` — TDD: non-Slack URL → skipped + no Authorization header sent + zero HTTP requests reach the mock server.
- `plugin/tests/slack-attachment-path-traversal.sh` — TDD: malformed `messageTs` (`../../etc`, `/etc`, embedded slash, no-dot, non-digits) → returns `[]` + no file created outside `baseDir`.
- 9 new node:test unit cases in `plugin/bridge/slack/tests/attachments.test.ts` covering host allowlist (legitimate, insecure scheme, spoofed suffix, workspace subdomain accepted) + messageTs validation matrix.

Suite count: 167 → 169 (modulo 158's pending suite which lands in a different release).
