## 160 тАФ Fork-bomb resistance end-to-end (Layers E + F)

Closes the bug-0002 fork-bomb-resistance class. Layers A (per-wrapper self-throttle), C (`wow-locate` recursion guard), and D (no-recursive-stub lint) shipped in story 152. This release adds:

- **Layer E** тАФ PreToolUse hook that blocks `rm`+glob for non-M roles. Converts the silent CC-stall (a peer waiting indefinitely on a permission prompt it cannot answer) into an immediate `{decision: "block"}` JSON with remediation pointers (`find -delete`, single-file `rm -f`, or M-nudge bypass).
- **Layer F** тАФ Test-runner host isolation. Both `plugin/tests/run-all.sh` and root `tests/run-all.sh` are now thin outer wrappers that apply `ulimit -u "$WOW_TEST_PROC_BUDGET"` (best-effort, default 2000) and exec `plugin/scripts/run-all-sandbox.py`. The Python wrapper calls `os.setsid()` to become its own session leader and installs a fork-based reaper that sweeps the process group with TERMтЖТgraceтЖТKILL after the wrapper itself exits тАФ leaked grandchildren die without breaking rc propagation. Inside the sandbox, `tests/run-all-inner.sh` wraps every test in `timeout "$WOW_TEST_TIMEOUT_S" bash` (default 300s).

Result: a runaway test reaps its own subtree on cleanup AND errors against the per-budget instead of saturating the host's user-wide ~10k proc limit. Co-resident CC sessions on the same Mac stay alive.

### Consumer action

`/reload-plugins`. The new PreToolUse hook activates on next agent startup or `/reload-plugins` тАФ no peer restart needed unless an in-flight session must immediately get the new block behavior. Existing sessions continue with the old hook set until they reload.

Both ulimit budget and per-test timeout are env-overridable: `WOW_TEST_PROC_BUDGET=N bash tests/run-all.sh` to widen or narrow the proc budget; `WOW_TEST_TIMEOUT_S=N` to widen or narrow the per-test timeout. Without `timeout(1)` on PATH (degenerate environment), the per-test wrapper no-ops тАФ ulimit + session-reap still bound the worst case.
