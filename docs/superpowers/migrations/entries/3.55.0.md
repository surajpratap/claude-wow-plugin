# `3.54.0` ‚Üí `3.55.0`

## AHOD mode ("All Hands On Deck") ‚Äî first-class independent-owner team mode

A human-activated mode in which every agent ‚Äî M included ‚Äî owns one work item end-to-end in parallel: story stub (M) ‚Üí plan ‚Üí TDD implementation ‚Üí full gate ‚Üí self code-review ‚Üí PR, each in its own worktree. The serial relay (SD‚ÜíPP‚ÜíT) is suspended; questions still route through M only; the human's PR merge is the cross-check. Continuous: M reassigns from the agreed pool on each `pr-created` until the human revokes the mode.

### Added

- `commands/_ahod-doctrine.md` ‚Äî the complete AHOD rulebook (owner lifecycle, premise verification, question routing, refusal/override, suspended-vs-in-force lists, M's dual duty, stand-down).
- `commands/_meta/ahod.md` + `commands/_meta/ahod-off.md` ‚Äî the activation/deactivation meta commands (run in M's terminal; sprint-exclusivity guard built in).
- `scripts/wow-config.sh` ‚Äî get/set/del helper for the NEW `implementations/config.json` (`{"schema":1,"mode":"default"|"ahod","ahod":{activated_ts, assignments}}`); atomic writes, worktree-invariant root, `.mode` closed enum. The file is the new consolidated home for mode-ish state (other dot-files migrate in later versions).
- Bus event types: `ahod-kickoff`, `ahod-ack`, `ahod-stand-down`, `read-ahod-doctrine`.
- MCP server: `read-ahod-doctrine` auto-inject on `ahod-kickoff` (always) and on `story-created` / `compaction-occurred` while `mode=ahod` (mirrors the trigger's `to`); `ahod-kickoff` joins the `read-token-discipline` + `read-learnings` trigger sets.

### Changed

- MCP server: while `mode=ahod`, the `pr-created` ‚Üí `code-review-request` auto-inject routes to the PR author (exact agent ID) instead of `pair-programmer-*` (sprint integration-branch suppression still wins); the `story-created` ‚Üí `read-skill` (writing-plans) inject mirrors the dispatch's `to`.
- `scripts/startup/phase_env.sh` emits `env: mode=<default|ahod>` at boot; all five `_<role>-startup.md` files key on it.
- Role files: SD/PP/T/S gain an `# AHOD mode` pointer section; `manager.md` gains the M-side AHOD section (kickoff, dual duty, continuous reassignment, bridge routing, stand-down) plus a sprint-trigger exclusivity guard; `_agent-protocol.md` gains the event rows, the exact-ID dispatch note on `story-created`, an AHOD mode section, and `config.json` in the file layout.
