## 104


`sprint-finalize.sh` — whole-sprint integration→main finalization helper (Story 104,
sprint 2026-05-18-sprint-infra-hardening). Closes the gap left by the per-item
`sprint-merge-bump.sh`: the 2026-05-17 sprint had no helper for the whole-sprint
finalization (M did v3.24.0 by hand; a `main`↔integration divergence caused the
PR-#101 merge conflict). **ADDED** `plugin/scripts/sprint-finalize.sh` — `--target
<X.Y.Z> [--dry-run]`. Runs from the integration-branch checkout and performs, in
order: (1) `git fetch origin <canonical>` (fatal-on-failure) + `git rebase
origin/<canonical>` with `--abort` on conflict; (2) resolve `3.24.0`/`3.24.1`
placeholders in every `entries/NEXT-*.md` against the canonical-branch version /
`--target`; (3) consolidate the resolved per-story entries (sorted) into one
`entries/<target>.md` — single `# \`<from>\` → \`<to>\`` header, each story body under
a `## <story-id>` subheading with its source `# …` header stripped — and `git rm -f`
the source `NEXT-*.md` files; (4) stamp `plugin.json` `.version` + the
`_manager-startup.md` "M targets plugin version" literal (reuses
`sprint-merge-bump.sh`'s verified single-quoted-regex `sed` pattern verbatim); (5)
pre-flight `version-coherence.sh` + `migration-entries-coherence.sh`; (6) commit
`sprint: finalize <sprint-id> → v<target>`. Idempotent: a re-run on a genuinely-
finalized branch (no NEXT-*.md remain AND version == target AND working tree clean
AND `origin/<canonical>` is an ancestor of HEAD) exits 0 as a no-op; a main-advanced
state re-rebases; a step-(e)-aborted (resolves/stamps staged-but-uncommitted) state
is NOT no-op'd. `--dry-run` gates every git + file mutation (rebase, sed-resolves,
consolidated-entry write, `git rm`s, the two stamps, the commit) to `would: …`
print-only — no working-tree write, no index change, no rebase-in-progress.
`CANONICAL_BRANCH` uses the assign-then-default form (`VAR=$(...); VAR="${VAR:-main}"`)
— the broken `… | sed … || echo main` pipeline that `sprint-merge-bump.sh:86`
carries (`sed` exits 0 on empty input → fallback never fires → `CANONICAL_BRANCH`
empty) is flagged to M as a latent bug, NOT fixed by this story. Preconditions:
working tree clean, HEAD matches `^sprint/` (refuses on `feat/*` to prevent a
destructive consolidate/stamp/commit on a per-item branch), `--target` strictly
greater than the canonical version by a portable component-wise compare (no
`sort -V` BSD/GNU divergence). **ADDED** `plugin/tests/sprint-finalize-helper.sh`
— 26 assertions across happy path, dry-run write-free (HEAD sha untouched, tree
clean, no `.git/rebase-merge`), idempotent re-run, conflict-pre-rebase
(rebase-abort + non-zero exit + no rebase-in-progress), malformed `--target`,
missing `--target`, wrong-branch refusal, and helper-level `_semver_cmp`
gt/eq/lt. Bundled bash test-suite count +1. **Consumer action after upgrade:**
`claude plugin update claude-wow`, `/reload-plugins`, restart peers. Just update
`.version`.
