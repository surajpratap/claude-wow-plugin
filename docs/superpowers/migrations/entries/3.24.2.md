The flat-compare form `.in_reply_to >= $cutoff` is an always-true bug (`dict >= "<string>"` in jq returns true because objects sort after strings). **FINDING-32 amendment** to Story 108 — see `plugin/tests/manager-pre-sleep-liveness.sh` case (g) for the anti-revert pin.
