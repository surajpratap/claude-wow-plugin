## 153 — Slack bridge silently drops its own bot's messages

### Added

- **`plugin/bridge/slack/src/bridge/bot-identity.ts`** — new helper exporting `captureBotIdentity(client): {userId, botId, authResp}` and `eventIsFromOwnBot(event, identity): boolean`. Single source of truth for own-bot detection across all 4 inbound event shapes (regular `message`, `bot_message` subtype, `message_changed`, `message_deleted` + reactions).
- **`plugin/bridge/slack/tests/own-bot-filter.test.ts`** — node:test unit cases (13 assertions) for the 4-predicate truth table + `captureBotIdentity` happy / missing-user_id / missing-bot_id paths.
- **`plugin/tests/slack-bridge-ignore-own-bot.sh`** — bash-driven synthetic-fixture test (12 assertions) mirroring `eventIsFromOwnBot` at the bash level + S's matched-fail-closed-exit parse path. Cases: another user's message forwarded; own-bot message dropped; message_changed for own-bot author dropped; own-bot reaction dropped; another user's reaction on our message forwarded; auth.test-failed line parses to `auth.test failed:` cause-detail; existing workspace-mismatch + missing-scope prefixes still parse; bot_message with matching / different bot_id; slacker.md line 152 contains the new prefix.

### Modified

- **`plugin/bridge/slack/src/index.ts`** — replaced the bare `app.client.auth.test()` + throw-on-no-user_id with a `captureBotIdentity` call wrapped in try/catch. On failure, calls `failClosedExit('[claude-slack-bridge] auth.test failed: <msg> — exiting')` — same fail-closed-exit shape as the existing `assertWorkspace` (story 092) and `assertScopes` (story 095) paths. The full `auth.test` response (`authResp`) flows through unchanged to 092 + 095 callers; no regression risk.
- **`plugin/bridge/slack/src/bridge/handlers.ts`** — `registerHandlers` signature changed from `botUserId: string` to `identity: BotIdentity` (the `{userId, botId, authResp}` triple). All 4 inbound event handlers (`app_mention`, `message`, `reaction_added`, `reaction_removed`) now short-circuit via `if (eventIsFromOwnBot(e, identity)) return;` at the top. The `message` handler's previous partial filter (lines 86-93, covered `bot_message` + bot-user-id but missed `message_changed` / `message_deleted`) is replaced by the unified helper. `app_mention` previously had no filter.
- **`plugin/commands/slacker.md`** line 152 — "Matched fail-closed exit" prefix list extends to 3 entries: workspace mismatch, missing OAuth scope(s), and `auth.test failed:`. S parses the cause-detail (line minus `[claude-slack-bridge] ` prefix and ` — exiting` suffix) and emits one `bridge-status: stopped` with `reason: "auth.test failed: <msg>"`.

### Why

The Slack bridge previously had partial own-bot filtering — `handlers.ts:86-93` covered some `message` shapes; `app_mention` had no filter at all; `message_changed` / `message_deleted` used the wrong field (`e.user` is the editor, not the original author). Result: in worst case, the bot's own messages re-entered the bus as inbound events and CC processed them again. The unified `eventIsFromOwnBot` helper closes all gaps.

The `auth.test` failure path previously threw a bare `Error` that crashed the bridge process with no S-side cause attribution. Routing through the existing `failClosedExit` pattern + extended `slacker.md` matched-prefix list gives S a clean cause-named escalation through the existing repair-via-restart-bridge nudge flow.

`/reload-plugins` + restart Slacker per standard upgrade hygiene.
