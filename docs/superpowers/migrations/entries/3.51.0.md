## 183 — idle-monitor respects a verify busy-marker (closes backlog-181)

The idle-monitor's `recent_bg_busy` was a 20-min time-bound heuristic; a verify longer than 20 min expired the busy window mid-run → false `all-idle-nudge`. Now the test-runner wrapper writes a lifecycle-tied PID-keyed busy-marker and the idle-monitor treats a live-PID marker as busy (no timeout). Closes backlog-181 (precise bg-child-PID liveness).

### Added

- `scripts/wow-process/idle-monitor.py` `has_live_verify_marker()` — a cohort is busy if any `implementations/.verify-running/<pid>.json` marker has a live PID (primary signal, no timeout); dead-PID markers are ignored + swept; a static `heartbeat_ts` is a secondary >6h PID-reuse guard.
- `plugin/tests/idle-monitor-verify-marker.sh` — 4-case red-green (live→busy, dead→idle+swept, >20-min regression, no-marker→idle) + `.red-without/183-verify-marker-busy.patch`.

### Modified

- `scripts/run-all-sandbox.py` — writes `implementations/.verify-running/<pid>.json` (`{pid, role, started_ts, heartbeat_ts}`) on entry (resolved via `CLAUDE_PROJECT_DIR`→walk-up, matching the idle-monitor's reader so a worktree-run verify still lands the marker where the reader looks); removes it on every parent-exit path (atexit + signal handler; a SIGKILL-leaked marker is swept by the dead-PID guard).
- `idle-monitor.py` `check_predicate` — a live `.verify-running` marker forces `busy`; `recent_bg_busy` kept as a complementary lower bound.

**Consumer action:** `/reload-plugins` to load the updated idle-monitor + run-all wrapper (a running idle-monitor picks it up after reload/restart). Suite count +1 (`idle-monitor-verify-marker.sh`). No role-doctrine change.
