## 169 ‚Äî Mechanize "show me the RED": `# RED-WITHOUT:` annotation + a revert-flips lint

Turns the sprint's highest-leverage practice ‚Äî a behavioral test guarding a subtle
gate must demonstrably fail when the fix is reverted ‚Äî from prose-diligence into a
mechanical pre-merge gate. A behavioral test carries:

```
# RED-WITHOUT: patch <relpath-under-tests/.red-without> -> <assert-spec>
```

The new `plugin/tests/red-without-lint.sh` parses each annotation, copies the
plugin tree to an isolated scratch, `git apply`s the named revert there, re-runs
ONLY the annotated test from the scratch, and asserts it flips RED with
`<assert-spec>` in the failure output ‚Äî then discards the scratch (the live tree is
never touched). A revert that leaves the test green is a hollow/no-op annotation ‚Üí
the lint FAILS naming it. Presence-grep is forbidden. The lint is self-hosting
(its own `# RED-WITHOUT:` proves it would catch its own degradation to a
presence-grep) and is auto-discovered by `run-all-inner.sh`'s `*.sh` glob.

### Added
- `plugin/tests/red-without-lint.sh` ‚Äî the verifier (grammar in its header = single source of truth).
- `plugin/tests/fixtures/red-without/` ‚Äî `parse/`, `good/`, `noop/`, `exclude/`, `missing/`, `allowlisted/` fixtures + `patches/`.
- `plugin/tests/.red-without/lint-selfhost.patch` ‚Äî the lint's self-host revert (assert-RED ‚Üí presence-grep).
- `plugin/tests/.red-without/grandfathered.txt` ‚Äî allowlist of current un-annotated behavioral tests (a NEW behavioral test cannot ship un-annotated; the existing backlog is reported, not failed).
- `plugin/tests/.red-without/ratelimit-backoff-case-f.patch` ‚Äî the case-f exemplar revert (reverts run.py's R2-1 `finally` cursor-persist).
