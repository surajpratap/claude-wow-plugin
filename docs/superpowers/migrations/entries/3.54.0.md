## 186 — `manager_monitor` rename + reactive 95% usage → M → urgent stop

M's `idle-monitor` daemon is generalized into `manager-monitor` — one ~60s loop with two named concerns (`idle_concern` + `usage_concern`). The usage concern now **pipes a usage signal to M** (Monitor→M channel) instead of emitting `pause`/`resume`/`escalate` to the bus itself, so **M owns the reaction**: an urgent, high-priority `pause` that explicitly tells peers to **kill their subagents**, plus a human escalation. The 5h pause trigger drops 98 → **95%**.

### Renamed

- `scripts/wow-process/idle-monitor.py` → `manager-monitor.py`; `idle-monitor.sh` → `manager-monitor.sh` (pidfile `manager-monitor-<role>.pid`, daemon `from:` prefix `manager-monitor-<pid>`).
- Monitor purpose `idle-monitor` → `manager-monitor` (`role-process-map.json`, `monitor-spec.sh`, `monitor-pipe.{sh,py}`, and the load-bearing arming in `scripts/startup/phase_bootstrap.sh`); events dir `.monitor-events/manager-monitor/`.
- Tests `idle-monitor-*.sh` → `manager-monitor-*.sh`. Stale references in `mcp/claude-wow-server/server.py`, `scripts/run-all-sandbox.py`, `scripts/hooks/log-activity.sh` updated. The idle **event-type names `all-idle-nudge` + `wake` are unchanged**.

### Added / changed behavior

- `manager-monitor.py`: `FIVE_HOUR_PAUSE_THRESHOLD` 98 → 95; `usage_concern()` pipes `usage-limit` / `usage-reset` / `usage-escalate` JSONL to stdout (no bus directive); `idle_concern()` factors the existing all-idle/wake + verify-marker logic.
- `commands/manager.md`: a `manager_monitor` usage-concern handler — on `usage-limit`, M broadcasts an urgent `pause` `{directive:"pause", priority:"urgent", kill_subagents:true}` + escalates the human; on `usage-reset`, `resume`; on `usage-escalate` (7d), notify-only. M never kills its own `manager-monitor`/`bus-tail`.
- `commands/_agent-protocol.md` + the 4 role files: the `pause` directive gains an optional `kill_subagents` field. On `kill_subagents:true`, a peer `TaskStop`s its spawned Agent subagents + ephemeral work-Monitors (tracking their task IDs) before halting — **but never its own bus-tail** (the resume lifeline). Absent/false → today's behavior (backward-compatible).
