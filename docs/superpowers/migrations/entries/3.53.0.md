# `3.52.0` → `3.53.0`

## 185 — statusline usage-compat verify at M startup

The usage auto-pause chain (statusline wrapper → `five-hour-usage.json` → monitor) fails **silently** when the user's statusline doesn't emit `rate_limits`: no state is ever persisted, the limit never fires, yet the user opted in. M now runs a one-shot, opt-in-gated chain verify at startup that confirms the wrapper is installed + wired + actually persists, and that the user's own statusline exposes `rate_limits` — surfacing an inert configuration with actionable guidance instead of a silent no-op.

### Added

- `scripts/wow-process/statusline-usage-verify.sh` — end-to-end verify. Prints `{healthy, checks:{installed,wired,persist_ok,statusline_emits_rate_limits}, reason}`; exit 0 = healthy. `persist_ok` feeds the generated wrapper a `rate_limits` fixture against an isolated `WOW_USAGE_STATE_FILE` and asserts it persisted; `statusline_emits_rate_limits` runs the user's recorded original statusline and classifies by **output** (`rate_limits` present → true / absent → false / empty-or-un-probable → `null` skip). Both checks run the user's statusline, so each is bounded by an **explicit** timeout branch (never the `A && timeout || fallback` idiom that re-runs unbounded on a 124 kill) — a hanging statusline cannot hang M startup. Health = `installed && wired && persist_ok && (statusline_emits_rate_limits != false)`.
- `plugin/tests/statusline-usage-verify.sh` — 5-case red-green (healthy / wrapper-missing / no-rate-limits / persist-unwritable / slow-orig-bounded) + `.red-without/185-verify-always-healthy.patch`. The slow-orig case proves the timeout genuinely bounds the run.

### Modified

- `commands/_manager-startup.md` — adds the "Usage auto-pause self-check" step after the opt-in section, gated on `usage_autopause` / `WOW_USAGE_AUTOPAUSE`. **Non-fatal + fail-open**: startup always continues; on a non-zero verify it surfaces one `AskUserQuestion` matching the failed check (re-install / disable opt-in / fix statusline). A `null` emit-check is treated as a pass.

**Consumer action:** `/reload-plugins` + restart M to load the new helper + the updated startup doctrine. Opt-in only — no effect unless usage auto-pause is enabled. Suite count +1 (`statusline-usage-verify.sh`). Role-doctrine change: `commands/_manager-startup.md` (M re-reads on next startup).
