## 171 — external-review.sh: reviewer prompt via `--prompt-file`, not an argv string

`external-review.sh` gains a `--prompt-file <path>` route. The wrapper reads the
body with `PROMPT="$(cat "$path")"` (the file BYTES become a variable value,
which is NOT re-evaluated for command substitution when passed as `"$PROMPT"`
argv), so a backtick / `$(...)` in a plan or review prompt reaches the reviewer
as literal text. This kills the footgun where the **caller's own shell**
command-substitutes an inline `"<prompt>"` before the wrapper ever runs (the
abort that cost a 168-r2 round-trip). PP doctrine (`pair-programmer.md` AC#5)
switches to the file route; the positional `<prompt>` form is retained for
backward-compat, and the load-bearing `< /dev/null` stdin redirect is unchanged.
