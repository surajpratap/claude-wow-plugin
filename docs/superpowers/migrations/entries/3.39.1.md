# `3.39.0` → `3.39.1`

## 174 — fix self-inflicted test-isolation storm (bus-leak + daemon-leak)

Running `plugin/tests/run-all.sh` was polluting the consuming project's **real**
shared bus and leaking orphaned wow-process daemons — the recurring phantom
"storm." Two facets, one root (tests under-isolated what they spawned):

- **Facet 1 — bus/marker leak.** Startup-invoking tests isolated only via
  `WOW_ROOT`, but the MCP server's `find_project_root()` never consulted
  `WOW_ROOT`; it resolved through `CLAUDE_PROJECT_DIR` else a cwd walk-up. The
  CLI `bus_emit` shim that `phase_bootstrap` (hello) / `phase_peer` (ping) call
  therefore fell through to the consumer's real `implementations/.message-bus.jsonl`.
- **Facet 2 — daemon leak.** 11 tests spawned real `bus-tail.sh` / `idle-monitor.py`
  daemons with incomplete reaping (SIGKILL-no-wait, no `EXIT` trap), so the
  outer `timeout` SIGTERM stranded them as orphans that accumulated across runs.

After this fix a full `run-all.sh` adds **zero** lines to the real bus, writes
**zero** markers into the real `.claude/.session-role-by-claude-pid/`, and leaves
**zero** net-new daemons.

### Added
- `plugin/tests/agent-boot-real-bus-untouched.sh` — combined storm-guard:
  facet-1 resolver-probe (a `WOW_ROOT`-only emit must land in its fixture, not a
  catch-all) + facet-2 daemon canary (spawn the real `idle-monitor.sh`, reap it,
  assert zero net-new non-main daemons) + standing proof the real bus + marker
  dir are byte-unchanged.
- `plugin/tests/.red-without/resolver-wow-root-first.patch`,
  `plugin/tests/.red-without/daemon-reaping-canary.patch` — each independently
  flips the guard RED (proves it is not a tautology).

### Modified
- `plugin/mcp/claude-wow-server/server.py` — `find_project_root()` honors
  `WOW_ROOT` **first** (before `CLAUDE_PROJECT_DIR`, then cwd walk-up).
- `plugin/scripts/whats-my-role.sh` — role-marker dir keys off `WOW_ROOT` when
  set (test isolation), else a worktree-invariant `--git-common-dir` fallback
  (never `--show-toplevel`).
- 10 startup leaker tests — pin `CLAUDE_PROJECT_DIR` beside `WOW_ROOT`.
- 11 daemon-spawning tests — canonical `trap … EXIT INT TERM` reaping idiom
  (kill wrapper-first, then temp-dir-anchored poll-sweep).

Suite count: +1 plugin suite.
